## Phase 5: MTP / Island Boy Architecture [NEXT]

### Key Insight: Memory Bandwidth Is The Bottleneck
35B MoE model = 10.7GB weights. DDR5 ~50GB/s → 214ms minimum per forward.
Current decode 1.5s = 7× memory bandwidth limit = poor cache utilization.

### "Island Boy" Pattern
Instead of streaming 40 layers × 1 token (10.7GB reads per step):
1. **Batch tokens through each layer** — process B=4 tokens through same weights
   - Weights loaded ONCE for 4 tokens → 4× throughput for 1.1× weight read time
2. **5-token startup lag** — first 5 tokens build KV cache + SSM states
   - Subsequent tokens: KV cache pre-populated, only need 1 new K/V per step
3. **Cache the working set** — keep hot layers in L2/L3 cache
   - Q5_K/Q6_K shared expert weights are small enough (~100MB) to stay cached

### MTP Speculative Decode (2-3× speedup)
**Draft phase** (blk.40, 1 layer):
  - Forward hidden_39 through blk.40 → predict token t+1
  - Embed token t+1, forward through blk.40 → predict token t+2
  - Repeat for N=3-4 drafts (each ~15ms = 60ms total)

**Verification phase** (full 40L, B=4):
  - Forward all 4 candidate tokens through layers 0-39 in a batch
  - Weight-load shared: B=4 reads same weights once
  - Accept longest prefix matching greedy target distribution

**MTP head structure** (blk.40):
  - `nextn.eh_proj.weight` [D_MODEL, vocab_size] — embed head projection
  - `nextn.enorm.weight` [D_MODEL] — embedding norm  
  - `nextn.hnorm.weight` [D_MODEL] — hidden norm
  - `nextn.shared_head_norm.weight` [D_MODEL] — shared head norm
  - Standard GQA attn_q/k/v + MoE (8 routed + 1 shared)
